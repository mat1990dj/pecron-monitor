# Pecron WB12200 — 12V Smart Battery / BMS

The WB12200 is a **12V 200Ah LiFePO4 battery** with a built-in Battery Management System (BMS). Unlike Pecron's power stations (E1500, E3600, E3800), it has **no AC inverter** — it's a pure 12V DC battery designed for:

- 🚐 **RVs and Motorhomes** — house battery replacement
- ☀️ **Off-grid solar systems** — 12V battery bank
- ⛵ **Marine applications** — trolling motor, electronics
- 🏕️ **Camping and overlanding** — portable 12V power

## Sensors

| Sensor | Description | Unit |
|--------|-------------|------|
| Battery % | State of charge | % |
| Voltage | Pack voltage (typical 10.4V–14.6V) | V |
| Current | Charge/discharge current (negative = discharge) | A |
| Temperature | Battery cell temperature | °C |
| DC Input Power | Charging power from solar/alternator | W |
| DC Output Power | Power being drawn by loads | W |
| Device Status | Shut Down / Charging / DC Discharge / Standby / Conservation | — |
| Fault Alarm | 18+ fault conditions (overvoltage, overcurrent, short circuit, etc.) | — |
| Remaining Time | Estimated time until empty (discharge) | H:M |
| Remaining Charging Time | Estimated time until full | H:M |

## Configurable Settings

These are **hidden settings** not documented in Pecron's official materials but exposed via the device's TSL (Thing Specification Language). You can change them via `--control`:

### Charge Limit Voltage
Controls when charging stops. Higher = more capacity, lower = longer cell life.

| Value | Voltage | Notes |
|-------|---------|-------|
| 0 | 12.8V | Very conservative |
| 1 | 13.2V | |
| 2 | 13.6V | |
| 3 | 14.0V | |
| 4 | 14.4V | Standard LiFePO4 |
| 5 | 14.6V | **Maximum** (default) |

### Discharge Limit Voltage
Low-voltage cutoff. Protects cells from deep discharge damage.

| Value | Voltage | Notes |
|-------|---------|-------|
| 0 | 10.4V | Deep discharge (not recommended) |
| 1 | 10.8V | |
| 2 | 11.2V | **Default** — good balance |
| 3 | 11.6V | Conservative |
| 4 | 12.0V | Very conservative |

### Charge Current Limit
Maximum charging current accepted.

| Value | Current | Notes |
|-------|---------|-------|
| 0 | 50A | |
| 1 | 60A | |
| 2 | 70A | |
| 3 | 80A | |
| 4 | 90A | |
| 5 | 100A | **Default** |
| 6 | 110A | |
| 7 | 120A | ⚡ Overclocked |

### Discharge Current Limit
Maximum current the battery will deliver. **220A on a 12V battery = 2,640W!**

| Value | Current | Notes |
|-------|---------|-------|
| 0 | 80A | |
| 1 | 100A | |
| 2 | 120A | |
| 3 | 140A | |
| 4 | 160A | |
| 5 | 180A | |
| 6 | 200A | **Default** |
| 7 | 220A | ⚡ Overclocked |

### Battery Heating Mode 🔥
**Critical for cold-weather RV/motorhome use.** LiFePO4 cells can be damaged by charging below freezing.

| Value | Mode | Description |
|-------|------|-------------|
| 0 | Charging Heating | Heats cells only when actively charging (saves power) |
| 1 | Keep Warm | Maintains cell temperature continuously (use when overnight temps drop below freezing and you need the battery ready at dawn) |

### Other Settings

| Setting | Options | Description |
|---------|---------|-------------|
| Standby Time | 24h / 48h / 7d / 14d / Always On | Auto-shutdown when idle |
| Battery Coding | 1–9, Host | For multi-battery pairing (which battery is which) |
| Buzzer | On / Off | Audible alerts |
| Power Indicator | On / Off | LED indicator light |

## Configuration

```yaml
# config.yaml
devices:
  - product_key: "p11vGo"
    device_key: "YOUR_DEVICE_KEY"
    name: "WB12200"
    auth_key: "YOUR_AUTH_KEY"
    lan_ip: "192.168.x.x"      # Optional: device's local IP for LAN polling
```

Run `python3 pecron_monitor.py --setup` to auto-discover your WB12200.

## Home Assistant Integration

When running with `--homeassistant`, the WB12200 exposes these HA entities:

- **Sensors:** Battery %, Voltage, Current, Temperature, DC Input/Output Power, Device Status, Fault Alarm, Charge/Discharge Voltage Limits, Current Limits, Heating Mode, Standby Time
- **Switches:** Buzzer, Power Indicator
- **Binary Sensor:** Expansion Pack (connected/disconnected)

All enum values are decoded to human-readable labels (e.g., "14.6V" instead of "5").

## Example: Changing Settings

```bash
# Set discharge cutoff to 11.6V (more conservative, better for cell longevity)
python3 pecron_monitor.py --control discharge_limiting_voltage_iaos 3

# Enable keep-warm mode for cold nights
python3 pecron_monitor.py --control battery_heating_mode_us 1

# Set charge current to 80A (gentler on cells)
python3 pecron_monitor.py --control charging_current_limit_iaos 3
```

## Notes

- The WB12200 has **no AC ports** — `ac_switch_hm`, `ups_status_hm`, and AC-related sensors will not apply.
- `remain_time` and `remain_charging_time` show as **N/A** when not actively charging/discharging (the firmware reports 65535 as a sentinel value).
- Current is negative when discharging, positive when charging.
- The `host_packet_current` value can be very precise (e.g., -0.067A) — useful for detecting parasitic draw in an RV.
