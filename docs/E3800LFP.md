# Pecron E3800LFP — High-Capacity Power Station

The E3800LFP is Pecron's **flagship power station** — a 3,840Wh LiFePO4 unit with 3,600W continuous AC output. Built for whole-home backup, off-grid living, and serious power needs.

## Specs

- **Capacity:** 3,840Wh (LiFePO4)
- **AC Output:** 3,600W continuous (pure sine wave), 120V/60Hz default
- **DC Output:** USB-A, USB-C (PD), 12V barrel (center positive ⊕)
- **DC/Solar Input:** Up to ~2,400W
- **AC Input:** Wall charging with adjustable power level
- **Expansion:** Supports expansion battery packs
- **Special Features:** 3 temperature sensors, eco mode, bypass mode, timed shutdown, touch panel lock

> **Note:** The DC barrel plug is **center positive** (tip = +, ring = −). Verified with multimeter.

## ⚠️ E3800 Firmware Quirks

The E3800 runs a **newer firmware revision** with significantly different behavior from the E1500/E3600:

### Local TCP: Settings Only
Local TCP connections return **only 14 settings fields** — battery %, switch states, and configuration values. **No voltage, temperature, or power data** via local TCP. This is a firmware design choice, not a bug.

### MQTT: Alternating Partial Packets
The E3800 sends telemetry via MQTT cloud in **3 alternating packet shapes**:
1. **Packet A:** Battery % only
2. **Packet B:** Temperature + input/output power
3. **Packet C:** Voltage (rare)

Each packet is incomplete. The monitor uses a **merge-and-cache strategy** — it accumulates fields across packet shapes so after 2-3 cycles you have complete data.

### High-Frequency Reporting
On startup, the monitor enables `high_frequency_reporting` to fill the cache faster (~30 seconds instead of ~3 minutes), then disables it after 60 seconds to reduce cloud traffic.

### SOC vs Host %
The E3800 reports two different battery percentages:
- **SOC %** — overall system state of charge (from non-host packets)
- **Host %** — host pack percentage (from `host_packet_data_jdb`)

When `host_packet_data_jdb` is present, `battery_percentage` mirrors host %, so the monitor separates these into distinct HA entities.

## Total PV Energy (Solar Tracking)

The E3800 exposes `total_energy` — a **lifetime cumulative solar generation counter** in kWh. This is tagged as `state_class: total_increasing` in Home Assistant, which means it integrates directly with HA's **Energy Dashboard** for tracking solar production over time.

## Home Assistant Integration

All sensors and switches above are auto-discovered. Notable HA features:

- **Energy Dashboard:** `total_energy` feeds directly into HA's energy tracking
- **3 Temperature Sensors:** Battery, charging plate, and inverter — great for monitoring thermals under heavy load
- **Eco Mode Switch:** Toggle quiet operation from HA
- **AC Charging Throttle:** Useful for load management — charge at 50% overnight to avoid tripping breakers

## Example Commands

```bash
# Enable eco/quiet mode
python3 pecron_monitor.py --control eco_quite_mode_as true

# Set AC charging to 50% (gentle overnight charging)
python3 pecron_monitor.py --control ac_charging_power_ios 5

# Lock touch panel (prevent accidental button presses)
python3 pecron_monitor.py --control device_touch_locking_as true

# Set UPS charge threshold to 80% (start charging when battery drops below 80%)
python3 pecron_monitor.py --control ups_start_charge_value_as 80

# Set AC output to 240V (for international use)
python3 pecron_monitor.py --control ac_output_voltage_io 5
```
